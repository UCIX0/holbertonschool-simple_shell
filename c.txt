4. Modo interactivo
    4.1. Imprimir el indicador "$"
        - Utilizar una función como 'printf' en C para mostrar el indicador "$" en la pantalla.
        --A continuación, se detallan los subpasos:
            4.1.1. Preparar el indicador de la línea de comandos
                - Establecer una cadena que contenga el símbolo "$" seguido de un espacio para mostrar el indicador.
            4.1.2. Utilizar una función de salida
                - Usar la función 'printf' en C u otra función similar para escribir el indicador en la pantalla, mostrándolo al usuario antes de aceptar la entrada.
            4.1.3. Asegurar que el indicador se muestre correctamente
                - Llamar a la función 'fflush' en C para asegurar que el indicador se muestre correctamente en la pantalla antes de recibir la entrada del usuario. Esto garantiza que la salida esté actualizada y el indicador sea visible incluso si no se ha alcanzado un punto de retorno automático.
    4.2. Leer la línea de comando del usuario
        4.2.1. Utilizar un búfer para leer muchos caracteres a la vez
            - Utilizar la función 'getline' en C, que lee la entrada del usuario y almacena el contenido en el búfer.
        --A continuación, se detallan los subpasos:
            4.2.1.1. Preparar el búfer
                - Asegurar que el búfer esté vacío y listo para recibir la entrada del usuario. Si es necesario, limpiar o reasignar el búfer para almacenar la nueva entrada.
            4.2.1.2. Leer la entrada del usuario
                - Utilizar la función 'getline' en C para leer la entrada del usuario desde la entrada estándar (stdin). La función 'getline' leerá caracteres hasta que se encuentre una nueva línea ('\n') o se alcance el final del archivo (EOF).
            4.2.1.3. Almacenar la entrada en el búfer
                - La función 'getline' almacena automáticamente la entrada del usuario en el búfer proporcionado, incluyendo el carácter de nueva línea ('\n').
            4.2.1.4. Ajustar el tamaño del búfer si es necesario
                - La función 'getline' ajustará el tamaño del búfer de forma dinámica si la entrada del usuario es más larga que el tamaño del búfer actual. Esto garantiza que la entrada completa del usuario se almacene en el búfer sin pérdida de información.
    4.3. Verificar si se ingresó Ctrl+D (fin de archivo)
        4.3.1. Comprobar el valor devuelto por 'getline'
            - Después de que la función 'getline' haya leído la entrada del usuario, verificar el valor que devuelve. Si 'getline' devuelve -1, esto indica que se ha encontrado el final del archivo (EOF) o que se ha ingresado Ctrl+D.
        4.3.2. Si se detecta Ctrl+D, salir de la shell
            4.3.2.1. Verificar si es necesario guardar el historial
                - Antes de salir de la shell, comprobar si el historial de comandos necesita ser guardado en un archivo, según las especificaciones del proyecto.
            4.3.2.2. Guardar el historial si es necesario
                - Si se requiere guardar el historial, escribir el contenido del historial en un archivo, como se especifica en el proyecto.
            4.3.2.3. Liberar recursos
                - Antes de terminar el programa, asegurarse de liberar todos los recursos utilizados, como el búfer, las estructuras de datos del historial y cualquier otra estructura dinámica que se haya utilizado.
            4.3.2.4. Terminar el programa
                - Una vez que se hayan liberado los recursos y se haya guardado el historial, terminar el programa utilizando una función como 'exit' en C.
    4.4. Verificar si se ingresó Ctrl+C
        - El manejo de Ctrl+C se configuró en el paso 1.5 (Control de señales), por lo que el programa no se cerrará automáticamente.
        - Si se detecta Ctrl+C, el programa simplemente imprimirá un nuevo indicador "$" y volverá al paso 4.1.
        --Aquí tienes un detalle adicional
            4.4.1. Recordar que el manejo de señales se configuró en el paso 1.5
                - En el paso 1.5, se estableció el manejo de la señal SIGINT (Ctrl+C) para evitar que el programa se cierre automáticamente cuando se reciba esta señal. En lugar de eso, se ejecutará una función personalizada que maneja la señal.
            4.4.2. Función personalizada para manejar Ctrl+C
                - La función personalizada que maneja la señal SIGINT no detendrá el programa. En su lugar, realizará las siguientes acciones:
                    4.4.2.1. Limpiar la entrada actual del usuario
                        - Si el usuario estaba escribiendo un comando, borrar la entrada para que no se procese más.
                    4.4.2.2. Imprimir un mensaje o una nueva línea
                        - Imprimir un mensaje, si es necesario, o simplemente una nueva línea para que el indicador "$" aparezca en una nueva línea.
            4.4.3. Volver al paso 4.1
                - Después de que la función personalizada haya manejado la señal SIGINT, el programa volverá al paso 4.1, que implica imprimir el indicador "$" y esperar una nueva entrada del usuario.
    4.5. Tokenizar la línea de comando
        4.5.1. Separar los comandos y argumentos
            - Utilizar una función o algoritmo adecuado (por ejemplo, 'strtok' en C o alguna otra función personalizada) para dividir la cadena en tokens, tomando en cuenta los delimitadores, como espacios y tabulaciones. Estos tokens representarán comandos y sus argumentos.
            - Almacenar estos tokens en una estructura de datos adecuada, como una lista o un arreglo, para su posterior procesamiento.
        4.5.2. Identificar los separadores de comandos (";") y operadores lógicos ("&&", "||")
            - A medida que se procesan los tokens, analizar cada token para identificar si corresponde a un separador de comandos (";") u operadores lógicos ("&&", "||").
            - Al encontrar un separador de comandos o un operador lógico, tomar nota de su posición en la estructura de datos que contiene los tokens y de qué tipo de separador u operador es.
            - Esta información será útil al momento de ejecutar los comandos en los pasos siguientes, ya que permitirá dividir la línea de comando en comandos individuales y establecer las condiciones para su ejecución basadas en los operadores lógicos.
    4.6. Verificar si se ingresaron comentarios (#)
        4.6.1. Buscar el carácter '#' en la línea de comando
            - Recorrer la línea de comando en busca del carácter '#', que indica el inicio de un comentario.
        4.6.2. Eliminar el comentario y el contenido posterior a él
            - Si se encuentra el carácter '#', eliminar el comentario y cualquier contenido que siga a este carácter en la línea de comando.
            - Esto se puede lograr cambiando el tamaño de la cadena o reemplazando el carácter '#' y los caracteres posteriores con el carácter nulo ('\0'), lo que efectivamente truncará la línea de comando en ese punto.
            - De esta manera, se ignorarán los comentarios al procesar los comandos y argumentos restantes en la línea de comando.
    4.7. Ejecutar el comando
        4.7.1. Determinar si el comando es un built-in
            - Comparar el comando con la lista de comandos built-in (por ejemplo, exit, env, setenv, unsetenv, cd, alias, history) para determinar si es uno de ellos.
        4.7.2. Si el comando es un built-in, ir al paso 5
            - Si se determina que el comando es un built-in, proceder al paso 5 para ejecutarlo y realizar las acciones correspondientes según el comando específico.
        4.7.3. Si no es un built-in, ir al paso 6
            - Si el comando no coincide con ninguno de los comandos built-in, se asume que es un comando externo.
            - Proceder al paso 6 para ejecutar el comando externo, lo que implica buscarlo en el PATH, crear un proceso hijo y ejecutarlo en ese proceso hijo.

5. Comandos built-in
    5.1. Identificar el comando built-in
        - A partir de la lista de comandos built-in (exit, env, setenv, unsetenv, cd, alias, history), determinar cuál es el comando que se ingresó y prepararse para ejecutar la función correspondiente.
        --Aquí se detalla el paso
            - A partir de la tokenización realizada en el paso 4.5, analizar el primer token (que correspondería al comando ingresado por el usuario).
            - Comparar el comando con la lista de comandos built-in (exit, env, setenv, unsetenv, cd, alias, history) utilizando una función como 'strcmp' en C o similar, para determinar si el comando ingresado coincide con alguno de los comandos built-in.
            - Si se encuentra una coincidencia, tomar nota del comando built-in específico que se ingresó y prepararse para ejecutar la función correspondiente en el siguiente paso (5.2. Ejecutar el comando built-in).
            - Si no se encuentra una coincidencia, continuar con el proceso de ejecución de comandos externos en el paso 6.
    5.2. Ejecutar el comando built-in
        - Dependiendo del comando built-in identificado en el paso 5.1, realizar la acción correspondiente:
            - exit: Salir de la shell, liberar recursos (como el búfer, las estructuras de datos y las variables utilizadas) y guardar el historial en el archivo .simple_shell_history en el directorio $HOME.
            - env: Iterar a través de las variables de entorno almacenadas en la copia del entorno creada en el paso 2.1 y mostrarlas en la salida estándar.
            - setenv: Aceptar un par nombre-valor como argumentos y establecer o modificar una variable de entorno en la copia del entorno almacenada en el paso 2.1.
            - unsetenv: Aceptar el nombre de una variable de entorno como argumento y eliminar dicha variable de la copia del entorno almacenada en el paso 2.1.
            - cd: Aceptar un argumento que indique el directorio de destino y cambiar el directorio de trabajo actual utilizando una función como 'chdir' en C.
            - alias: Aceptar un argumento que indique el nombre del alias y el comando asociado, y almacenar esa información en una estructura de datos (por ejemplo, un mapa o un diccionario) para su uso futuro.
            - history: Mostrar el historial de comandos ejecutados almacenado en la estructura de datos creada en el paso 1.4.
        --En general, los comandos built-in se ejecutan directamente en el proceso principal de la shell sin crear procesos hijos. Esto significa que no se requiere la creación de procesos adicionales para llevar a cabo las acciones de los comandos built-in.
    5.3. Volver al paso 4.1
        - Después de ejecutar el comando built-in y completar la acción correspondiente, la shell debe continuar en modo interactivo esperando nuevas entradas del usuario.
        - Para hacer esto, volver al paso 4.1, donde se muestra el indicador "$" en la pantalla, indicando que la shell está lista para recibir el siguiente comando del usuario.
        - La shell seguirá en este ciclo, leyendo la entrada del usuario, procesando y ejecutando comandos, hasta que se reciba una señal para salir de la shell (por ejemplo, al ingresar el comando built-in "exit" o al detectar Ctrl+D).
