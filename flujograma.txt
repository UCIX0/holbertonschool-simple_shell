			- Esto se puede lograr cambiando el tamaño de la cadena o reemplazando el carácter '#' y los caracteres posteriores con el carácter nulo ('\0'), lo que efectivamente truncará la línea de comando en ese punto.
			- De esta manera, se ignorarán los comentarios al procesar los comandos y argumentos restantes en la línea de comando.

	4.7. Ejecutar el comando

		4.7.1. Determinar si el comando es un built-in
			- Comparar el comando con la lista de comandos built-in (por ejemplo, exit, env, setenv, unsetenv, cd, alias, history) para determinar si es uno de ellos.

		4.7.2. Si el comando es un built-in, ir al paso 5
			- Si se determina que el comando es un built-in, proceder al paso 5 para ejecutarlo y realizar las acciones correspondientes según el comando específico.

		4.7.3. Si no es un built-in, ir al paso 6
			- Si el comando no coincide con ninguno de los comandos built-in, se asume que es un comando externo.
			- Proceder al paso 6 para ejecutar el comando externo, lo que implica buscarlo en el PATH, crear un proceso hijo y ejecutarlo en ese proceso hijo.

5. Comandos built-in
	5.1. Identificar el comando built-in
		- A partir de la lista de comandos built-in (exit, env, setenv, unsetenv, cd, alias, history), determinar cuál es el comando que se ingresó y prepararse para ejecutar la función correspondiente.

		--Aquí se detalla el paso

			- A partir de la tokenización realizada en el paso 4.5, analizar el primer token (que correspondería al comando ingresado por el usuario).
			- Comparar el comando con la lista de comandos built-in (exit, env, setenv, unsetenv, cd, alias, history) utilizando una función como 'strcmp' en C o similar, para determinar si el comando ingresado coincide con alguno de los comandos built-in.
			- Si se encuentra una coincidencia, tomar nota del comando built-in específico que se ingresó y prepararse para ejecutar la función correspondiente en el siguiente paso (5.2. Ejecutar el comando built-in).
			- Si no se encuentra una coincidencia, continuar con el proceso de ejecución de comandos externos en el paso 6.

	5.2. Ejecutar el comando built-in

		- Dependiendo del comando built-in identificado en el paso 5.1, realizar la acción correspondiente:
			- exit: Salir de la shell, liberar recursos (como el búfer, las estructuras de datos y las variables utilizadas) y guardar el historial en el archivo .simple_shell_history en el directorio $HOME.
			- env: Iterar a través de las variables de entorno almacenadas en la copia del entorno creada en el paso 2.1 y mostrarlas en la salida estándar.
			- setenv: Aceptar un par nombre-valor como argumentos y establecer o modificar una variable de entorno en la copia del entorno almacenada en el paso 2.1.
			- unsetenv: Aceptar el nombre de una variable de entorno como argumento y eliminar dicha variable de la copia del entorno almacenada en el paso 2.1.
			- cd: Aceptar un argumento que indique el directorio de destino y cambiar el directorio de trabajo actual utilizando una función como 'chdir' en C.
			- alias: Aceptar un argumento que indique el nombre del alias y el comando asociado, y almacenar esa información en una estructura de datos (por ejemplo, un mapa o un diccionario) para su uso futuro.
			- history: Mostrar el historial de comandos ejecutados almacenado en la estructura de datos creada en el paso 1.4.

		--En general, los comandos built-in se ejecutan directamente en el proceso principal de la shell sin crear procesos hijos. Esto significa que no se requiere la creación de procesos adicionales para llevar a cabo las acciones de los comandos built-in.

	5.3. Volver al paso 4.1
		- Después de ejecutar el comando built-in y completar la acción correspondiente, la shell debe continuar en modo interactivo esperando nuevas entradas del usuario.
		- Para hacer esto, volver al paso 4.1, donde se muestra el indicador "$" en la pantalla, indicando que la shell está lista para recibir el siguiente comando del usuario.
		- La shell seguirá en este ciclo, leyendo la entrada del usuario, procesando y ejecutando comandos, hasta que se reciba una señal para salir de la shell (por ejemplo, al ingresar el comando built-in "exit" o al detectar Ctrl+D).

6. Comandos externos

	6.1. Verificar si el comando existe en el PATH
		- Buscar el comando en las rutas especificadas en la variable de entorno PATH.
		- Si se encuentra el comando en una de las rutas, continuar con el siguiente paso.
		- Si no se encuentra el comando, imprimir un mensaje de error informando al usuario que el comando no se encontró, y luego volver al paso 4.1 para esperar la siguiente entrada del usuario.

		--Aquí se detalla el paso

			6.1.1. Obtener la variable de entorno PATH
				- Utilizar una función como 'getenv' en C para obtener la cadena que contiene la variable de entorno PATH.

			6.1.2. Dividir la variable PATH en rutas individuales
				- Utilizar una función como 'strtok' en C para dividir la cadena PATH en rutas individuales, usando el carácter ':' como delimitador.

			6.1.3. Buscar el comando en cada ruta
				- Recorrer cada ruta obtenida en el paso 6.1.2 y verificar si el comando existe en esa ruta.
				- Utilizar funciones como 'access' en C para comprobar si el archivo del comando existe y si se puede ejecutar.

			6.1.4. Comprobar si se encontró el comando
				- Si se encuentra el comando en una de las rutas, continuar con el siguiente paso (6.2).
				- Si no se encuentra el comando en ninguna ruta, imprimir un mensaje de error informando al usuario que el comando no se encontró y luego volver al paso 4.1 para esperar la siguiente entrada del usuario.

	6.2. No llamar a fork si el comando no existe
		- Después de verificar si el comando existe en el PATH (paso 6.1) y si el resultado es negativo, es decir, el comando no se encuentra en el PATH, se debe evitar la creación de un proceso hijo.

		- En este caso, no se debe llamar a la función 'fork()' y no se debe ejecutar ningún proceso hijo para el comando que no existe.

		- En lugar de crear un proceso hijo, se debe mostrar el mensaje de error informando al usuario que el comando no se encontró.

		- Una vez mostrado el mensaje de error, el programa debe retornar al paso 4.1 (imprimir el indicador "$") para esperar la siguiente entrada del usuario en el modo interactivo.

	6.3. Crear un proceso hijo usando fork()
		- Antes de crear un proceso hijo, asegúrese de que el comando exista en el PATH (según el paso 6.1).

		- Utilice la función 'fork()' en C para duplicar el proceso actual. La función 'fork()' creará un nuevo proceso, denominado proceso hijo, que será una copia casi exacta del proceso en ejecución, llamado proceso padre.

		- La función 'fork()' devuelve un valor de tipo 'pid_t'. Si el valor devuelto es 0, indica que el código se está ejecutando en el proceso hijo. Si el valor es mayor a 0, indica que el código se está ejecutando en el proceso padre y el valor devuelto es el ID del proceso hijo. Si el valor es -1, significa que hubo un error al intentar crear el proceso hijo.

		- Después de llamar a 'fork()', tanto el proceso padre como el proceso hijo continuarán ejecutando el código desde el punto donde se llamó a 'fork()'. Por lo tanto, es necesario utilizar el valor devuelto por 'fork()' para determinar si el código se está ejecutando en el proceso padre o en el proceso hijo y proceder en consecuencia.

	6.4. Ejecutar el comando en el proceso hijo usando exec()
		- Una vez que se haya creado el proceso hijo con éxito (según el paso 6.3), el código en el proceso hijo debe proceder a ejecutar el comando externo.

		- Utilice una función de la familia 'exec()' en C, como 'execvp()' o 'execv()', para reemplazar la imagen del proceso hijo con la del comando externo. La función 'exec()' carga el programa especificado en el proceso hijo y lo ejecuta con los argumentos proporcionados.

		- 'execvp()' es útil porque busca el comando en las rutas especificadas en la variable de entorno PATH y permite pasar un conjunto de argumentos como un array de cadenas terminado en NULL.

		- 'execv()' requiere que se proporcione la ruta completa del archivo ejecutable y también permite pasar un conjunto de argumentos como un array de cadenas terminado en NULL.

		- Cuando 'exec()' se ejecuta correctamente, el proceso hijo se reemplaza completamente por el programa del comando externo y no regresa al código original de la shell. En caso de error, 'exec()' devuelve -1 y el proceso hijo debe finalizar adecuadamente, notificando al proceso padre del error si es necesario.

	6.5. Esperar a que el proceso hijo termine usando wait()
		- Después de crear el proceso hijo (paso 6.3) y de haber ejecutado el comando externo en el proceso hijo (paso 6.4), es necesario que el proceso padre (la shell) espere a que el proceso hijo finalice su ejecución.

		- Para lograr esto, se utiliza la función 'wait()' o una de sus variantes como 'waitpid' en el proceso padre. Al llamar a esta función, el proceso padre se bloquea temporalmente y espera a que el proceso hijo termine.

		- Cuando el proceso hijo finaliza su ejecución y sale, el sistema operativo envía una señal al proceso padre. En respuesta a esta señal, la función 'wait()' o 'waitpid' devuelve el control al proceso padre, permitiéndole continuar con su ejecución.

		- Al usar 'wait()' o 'waitpid', se asegura la sincronización adecuada entre el proceso padre y el hijo, evitando que el proceso padre avance hasta que el comando externo se haya ejecutado completamente en el proceso hijo.

	6.6. Volver al paso 4.1
		- Después de que el proceso hijo haya finalizado su ejecución y el proceso padre haya esperado correctamente (según el paso 6.5), el control regresa al proceso padre, es decir, a la shell.

		- En este punto, el proceso padre (la shell) debe regresar al paso 4.1, en el cual se muestra nuevamente el indicador "$" en la pantalla.

		- Con el indicador "$" impreso, la shell está lista para recibir la siguiente entrada del usuario en el modo interactivo, permitiendo así continuar con la ejecución de más comandos built-in o externos según las instrucciones del usuario.

7. Modo no interactivo
	7.1. Leer el script de entrada
		- Al iniciar el programa en modo no interactivo con un archivo de script como argumento, abrir el archivo utilizando una función como 'fopen' en C. Asegurarse de manejar posibles errores, como archivos no encontrados o problemas de permisos.

		- Leer el contenido del archivo de script línea por línea utilizando una función como 'fgets' en C. Esto permite procesar cada línea de comando individualmente.

		- Almacenar cada línea leída en una estructura de datos adecuada, como un búfer, una lista enlazada o un vector de cadenas, para su posterior procesamiento y ejecución en la shell. Asegurarse de liberar adecuadamente la memoria al finalizar la ejecución del script.

	7.2. Ejecutar el script siguiendo los pasos 4.2 a 6.6 según corresponda
		- Para cada línea leída del archivo de script, realizar los siguientes pasos:

			1. Tokenizar la línea de comando (paso 4.5).
				- Separar los comandos y argumentos, e identificar los separadores de comandos (";") y operadores lógicos ("&&", "||").

			2. Verificar si se ingresaron comentarios (#) y eliminarlos si es necesario (paso 4.6).
				- Buscar el carácter '#' en la línea de comando, y si se encuentra, eliminar el comentario y el contenido posterior a él.

			3. Verificar si la línea es un comentario completo, contiene operadores lógicos o separadores de comandos.
				- Si la línea es un comentario completo, continuar con la siguiente línea del script.
				- Si contiene operadores lógicos o separadores de comandos, procesarlos de manera adecuada, ejecutando los comandos en función de las condiciones lógicas o en secuencia.

			4. Determinar si el comando es un built-in o un comando externo (paso 4.7).
				- Verificar si el comando coincide con alguno de los comandos built-in (exit, env, setenv, unsetenv, cd, alias, history) o si es un comando externo.

			5. Ejecutar el comando correspondiente, ya sea un comando built-in (paso 5) o un comando externo (paso 6).

			6. Repetir este proceso hasta que se hayan ejecutado todas las líneas del script. Cuando todas las líneas del script hayan sido procesadas y ejecutadas, continuar con el paso 7.3 para salir de la shell en modo no interactivo.

	7.3. Salir de la shell

			1. Guardar el historial de comandos (si corresponde) en un archivo, como .simple_shell_history en el directorio $HOME. Este paso es opcional y puede depender de cómo esté configurada la shell.

			2. Cerrar el archivo del script utilizando una función como 'fclose' en C, si se abrió previamente con 'fopen'.

			3. Liberar los recursos utilizados, como búferes, estructuras de datos o memoria dinámica asignada durante la ejecución del programa. Esto incluye, por ejemplo, liberar memoria asignada para almacenar tokens, comandos, argumentos, etc.

			4. Terminar el programa de shell utilizando una función como 'exit' en C, con un código de salida apropiado (por lo general, 0 si no se produjeron errores o un valor diferente en caso de error). Esto permite una finalización ordenada del programa y la liberación de todos los recursos del sistema asociados con el proceso de la shell.

8. Salir
	8.1. Guardar el historial en .simple_shell_history en el directorio $HOME
		- Para guardar el historial de comandos en un archivo llamado .simple_shell_history en el directorio $HOME del usuario, seguir estos pasos:

			1. Obtener la ruta del directorio $HOME del usuario utilizando la función 'getenv' en C y la variable de entorno "HOME".

			2. Construir la ruta completa del archivo .simple_shell_history concatenando la ruta del directorio $HOME con el nombre del archivo.

			3. Utilizar funciones de E/S de archivos, como 'fopen' en C, para abrir o crear el archivo .simple_shell_history en modo de escritura ('w' o 'a' según corresponda).

			4. Verificar que el archivo se haya abierto correctamente. Si no se puede abrir el archivo, imprimir un mensaje de error y continuar con el siguiente paso de salida.

			5. Recorrer la estructura de datos del historial (por ejemplo, una lista enlazada, un vector o un arreglo dinámico de cadenas) y escribir cada comando en el archivo .simple_shell_history utilizando funciones como 'fprintf' en C.

			6. Después de recorrer y guardar todo el historial de comandos, cerrar el archivo utilizando la función 'fclose' en C.

	8.2. Liberar recursos y finalizar
		Antes de terminar el programa de shell, es crucial liberar todos los recursos utilizados durante la ejecución para evitar fugas de memoria y otros problemas. Aquí se detallan los pasos generales para hacerlo:

			1. Liberar búferes dinámicos: Si se han utilizado búferes dinámicos para almacenar, por ejemplo, la línea de comando del usuario, asegurarse de liberarlos utilizando la función 'free' en C antes de salir del programa.

			2. Liberar estructuras de datos: Si se han utilizado estructuras de datos, como listas enlazadas, vectores o arreglos dinámicos para almacenar información como el historial de comandos, liberar la memoria asignada para cada elemento y la estructura en sí utilizando la función 'free' en C.

			3. Cerrar archivos: Si se han abierto archivos a lo largo del programa, como el archivo .simple_shell_history, asegurarse de cerrarlos utilizando la función 'fclose' en C antes de salir.

			4. Liberar punteros: Si se han utilizado punteros a lo largo del programa para apuntar a recursos como variables de entorno o alias, liberar la memoria asignada a esos recursos y, si es necesario, a los punteros en sí.

			5. Terminar el programa: Una vez que se hayan liberado todos los recursos, utilizar una función como 'exit' en C para terminar el programa de shell. Proporcionar un código de salida adecuado, como EXIT_SUCCESS si el programa se ha ejecutado correctamente o EXIT_FAILURE si ha ocurrido un error.
